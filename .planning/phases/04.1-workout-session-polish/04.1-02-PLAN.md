**1d. Extend useTempoVoice to expose currentPhase and countdown:**

The user's locked decision requires "DOWN 3" to show as a live tempo phase + countdown in the center block. Currently `useTempoVoice` only exposes `isPlaying`, `startTempo`, `stopTempo`. Extend it:

Add state to track the current phase and countdown number:
```typescript
const [currentPhase, setCurrentPhase] = useState<string | null>(null); // 'DOWN', 'HOLD', 'UP', 'HOLD'
const [countdown, setCountdown] = useState<number>(0);
```

Update `playTempoCycle` to set these values at each step:
- Before `await speakAndWait('Down')`: call `setCurrentPhase('DOWN')` and `setCountdown(config.eccentric)`
- In the eccentric countdown loop, before each `await speakAndWait(i.toString())`: call `setCountdown(i)`
- Before `await speakAndWait('Hold')` (bottom): call `setCurrentPhase('HOLD')` and `setCountdown(config.pauseBottom)`
- Before `await speakAndWait('Up')`: call `setCurrentPhase('UP')` and `setCountdown(config.concentric)`
- In the concentric countdown loop: call `setCountdown(i)`
- Before hold at top: call `setCurrentPhase('HOLD')` and `setCountdown(config.pauseTop)`
- When stopping (stopTempo): call `setCurrentPhase(null)` and `setCountdown(0)`

Update the return to include `currentPhase` and `countdown`:
```typescript
return {
  startTempo,
  stopTempo,
  isPlaying,
  currentPhase,   // 'DOWN' | 'HOLD' | 'UP' | null
  countdown,       // current countdown number
};
```
