## Decisions Made

- `currentPhase` and `countdown` are React state in useTempoVoice (not refs) so the UI re-renders on each change — this is intentional for the live display use case
- `Animated.View` from react-native-reanimated wraps the completed sets expansion now, even though animations are Plan 04's focus — avoids import restructuring later
- Static tempo string format `3-1-2-0` (eccentric-pauseBottom-concentric-pauseTop) matches the existing SplitDetailScreen display convention
- Exercise counter (1/3) shown below exercise name, above tempo — provides context without cluttering
