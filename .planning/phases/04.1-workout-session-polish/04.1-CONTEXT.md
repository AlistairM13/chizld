# Phase 4.1: Workout Session Polish - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Improve the look and feel of the workout session screen — better visual hierarchy, refined input controls, and enhanced UX for the set logging flow. This is a UI polish phase, not new functionality.

</domain>

<decisions>
## Implementation Decisions

### Layout Philosophy
- Minimal, focused design — "one thing at a time"
- Two screen states: active set view and rest timer view
- Screen largely empty except for essential elements
- Black background throughout

### Active Set View Layout
- **Center block (dominant):** Exercise name + set counter + tempo countdown
  - Format: "BENCH PRESS (2/5)" with large "DOWN 3" below
  - Tempo countdown is the visual focus when active
- **Bottom row:** Weight, reps, RPE inputs in a horizontal row
- **Corner elements:** Small elapsed time timer, running XP counter
- **Edge navigation:** Small < > arrow buttons at screen edges for exercise nav
- **Bottom bar:** Full-width COMPLETE SET button

### Completed Sets Display
- Hidden by default — only show "3 sets complete" count
- Tap to expand if user wants to see history
- Focus stays on current set

### Input Controls
- Weight: Number display with +/- steppers (2.5kg increments)
- Reps: Steppers only (+/- buttons)
- RPE: Horizontal slider (6-10 scale) — more tactile than tappable buttons
- Feedback: Both haptic vibration AND visual pulse on value change

### Rest Timer View
- Full black screen — nothing else visible
- Large centered seconds countdown (just "45", not "0:45")
- Below countdown: -15s | SKIP | +15s button row
- When timer hits zero: Show "REST COMPLETE", user taps to return
- Fade transition between set view and rest view (not instant cut)

### Progress Visibility
- Exercise progress: Part of center block — "BENCH PRESS (2/5)"
- Set counter: Small "Set 3" label (location TBD by implementation)
- Elapsed time: Small corner timer running throughout
- XP: Running total counter visible, updates after each set

### Claude's Discretion
- Exact positioning of corner elements (elapsed time, XP counter)
- Animation timing for fade transitions
- Specific dimensions and spacing of input controls
- Font sizes within the established hierarchy

</decisions>

<specifics>
## Specific Ideas

- "The screen should be largely blank except the number of reps and tempo info"
- Tempo countdown should be large and dominant during active sets
- Rest screen should be completely black except the timer — creates a visual break
- Think of it as a "focused view" — one set at a time, no distractions

</specifics>
