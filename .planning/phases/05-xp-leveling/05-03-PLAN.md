**Update useSessionSummary hook:**

1. Accept additional params: `sessionId: string, zoneId: string`

2. Add query for XP breakdown from workout_sets:
   - Sum all XP components by re-calculating from set data
   - Query sets with tempo data to calculate tempo bonus

3. Query xp_history for this session to get actual persisted XP:
   ```sql
   SELECT SUM(xp_amount) as total_xp FROM xp_history WHERE source_id = ?
   ```

   If no xp_history records yet (session not finalized), calculate from sets.

4. Add `finalizeAndGetResult` function (called by summary screen):
   - Call useXPService.finalizeSession(zoneId, totalXP, sessionId)
   - Returns { leveledUp, newLevel }
   - Only call once (track with useRef)

5. Calculate xpBreakdown by iterating through sets:
   - base = totalSets * 10
   - volumeBonus = sum of floor(weight * reps / 100) for each set
   - prBonus = count of PR sets * 50 (need to query which sets were PRs)
   - consistencyBonus = 20 if zone was trained in 7 days before session (once per session, not per set)
   - tempoBonus = sum of (setXP * 0.5) for tempo sets

**Note on PR and tempo tracking:**
Currently workout_sets doesn't store isPR or tempoEnabled. Options:
A) Add columns to workout_sets (schema migration)
B) Calculate PR retroactively by comparing to historical data
C) Pass XP breakdown from WorkoutSessionScreen via route params

For v1 simplicity, use option C: Pass totalXP and breakdown from the session screen.
