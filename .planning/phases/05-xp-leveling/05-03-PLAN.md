Replace placeholder XP calculation with real XP from route params and add finalization logic.

**Update SessionSummary interface:**
```typescript
export interface SessionSummary {
  sessionId: string;
  startedAt: Date;
  endedAt: Date | null;
  duration: number;
  totalSets: number;
  exercises: ExerciseSummary[];
  totalVolume: number;
  xpEarned: number;         // Real XP from route params or recalculated
  xpBreakdown: {            // Detailed breakdown
    base: number;           // totalSets * 10
    volumeBonus: number;    // Sum of floor(weight * reps / 100) for each set
    prBonus: number;        // Count of PR sets * 50
    consistencyBonus: number; // 20 if applied (once per session, not per set)
    tempoBonus: number;     // Sum of (setXP * 0.5) for tempo sets
  };
  leveledUp: boolean;       // Did zone level up?
  newLevel: number | null;  // New level if leveled up
  zoneId: string;           // Zone for display
  zoneName: string;         // Zone name for display
}
```

**Update useSessionSummary hook:**

1. Accept params: `sessionId: string, zoneId: string, preCalculatedXP?: { total: number, breakdown: XPBreakdown }`

2. For active session (preCalculatedXP provided):
   - Use the pre-calculated values directly from route params
   - These were accumulated by WorkoutSessionScreen during the workout
   - Breakdown includes: base (sets * 10), volumeBonus, prBonus, consistencyBonus (20 if applied, 0 otherwise), tempoBonus

3. For historical session (no preCalculatedXP):
   - Query xp_history for this session: `SELECT xp_amount FROM xp_history WHERE source_id = ?`
   - Calculate breakdown approximation from sets (may not have PR/tempo info stored)
   - Note: This is best-effort for old sessions

4. Add `finalizeAndGetResult` function (called by summary screen):
   - Call useXPService.finalizeSession(zoneId, totalXP, sessionId)
   - Returns { leveledUp, newLevel }
   - Only call once (track with useRef hasFinalized)
   - Set leveledUp and newLevel in state

5. Query zone_stats for zoneName:
   ```sql
   SELECT name FROM zones WHERE id = ?
   ```

**Consistency bonus note:**
- The 20 XP consistency bonus is applied once per session (controlled by Plan 02)
- xpBreakdown.consistencyBonus will be either 20 or 0
- This is NOT totalSets * 20 - it's a single bonus for the session
