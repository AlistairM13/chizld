       - Remove `"expo-router"` from the plugins array (keep expo-splash-screen, expo-sqlite, expo-font)

       - Leave everything else unchanged (name, slug, icons, android config, etc.)

    **Entry point:**

    3. Create src/main.tsx:
       ```
       import { registerRootComponent } from 'expo';
       import App from './App';
       registerRootComponent(App);
       ```
       IMPORTANT: `export default` alone does NOT work with custom entry files. Must use registerRootComponent.

    **App root component:**

    4. Create src/App.tsx:
       - Import and call `SplashScreen.preventAutoHideAsync()` at MODULE LEVEL (outside component) -- this keeps the native splash visible during initialization
       - Load ALL fonts in a single `useFonts` call combining all three font packages:
         From @expo-google-fonts/chakra-petch: ChakraPetch_700Bold, ChakraPetch_600SemiBold, ChakraPetch_400Regular
         From @expo-google-fonts/barlow-condensed: BarlowCondensed_600SemiBold, BarlowCondensed_500Medium, BarlowCondensed_400Regular
         From @expo-google-fonts/jetbrains-mono: JetBrainsMono_700Bold, JetBrainsMono_400Regular
         Use `useFonts` from any of the @expo-google-fonts packages (they all re-export expo-font's useFonts)
       - Lock orientation in useEffect on mount: `ScreenOrientation.lockAsync(ScreenOrientation.OrientationLock.LANDSCAPE)`
       - Initialize MMKV defaults in useEffect on mount: call `initMMKVDefaults()` from src/services/storage.ts
       - Handle splash screen: when `fontsLoaded` is true, call `SplashScreen.hideAsync()`. Also handle error case from useFonts to prevent splash hanging forever.
       - Return `null` while fonts loading (keeps splash visible -- no white flash)
       - Component tree when ready:
         ```
         <SQLiteProvider databaseName="chizld.db" onInit={initDatabase}>
           <NavigationContainer>
             <RootNavigator />
           </NavigationContainer>
         </SQLiteProvider>
         ```
         SQLiteProvider MUST wrap NavigationContainer (outermost) so all screens can call useSQLiteContext().

    **Navigation:**

    5. Create src/navigation/types.ts:
       ```
       export type RootStackParamList = {
         Main: undefined;

       export type MainTabParamList = {
         Home: undefined;

         Fuel: undefined;
       };
       ```

    6. Create src/navigation/RootNavigator.tsx:

       - Single screen for now: "Main" component={MainTabs} with headerShown: false
       - Export as named export: `export function RootNavigator()`

    7. Create src/navigation/MainTabs.tsx:

       - 4 tabs: Home (CharacterScreen), Train (WorkoutListScreen), Sleep (SleepDashboardScreen), Fuel (DailyViewScreen)
       - Tab bar styling (Cyberpunk HUD aesthetic):
         - backgroundColor: colors.bg.primary ('#0A0A0F')
         - borderTopColor: colors.zone.cold ('#2A2A3A')
         - borderTopWidth: 1
         - height: 52
         - tabBarActiveTintColor: colors.ember[500] ('#FF8C1A')
         - tabBarInactiveTintColor: colors.text.muted ('#555566')
         - tabBarLabelStyle: { fontFamily: fonts.heading (BarlowCondensed_600SemiBold), fontSize: 10, letterSpacing: 1, textTransform: 'uppercase' }
         - headerShown: false on all screens
       - Tab bar icons: Use MaterialCommunityIcons from @expo/vector-icons (included with Expo, no install needed)
         - Home: 'human' icon
         - Train: 'dumbbell' icon
         - Sleep: 'moon-waning-crescent' icon
         - Fuel: 'fire' icon
         - Icon size: 22
       - Import colors from src/constants/colors.ts and fonts from src/constants/fonts.ts
