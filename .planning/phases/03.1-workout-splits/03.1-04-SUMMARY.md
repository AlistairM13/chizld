# Phase 3.1 Plan 04: Split Detail Screen Summary

Split-sourced workout start with primary zone attribution.

## What Was Built

**SplitDetailScreen** (`src/screens/workout/SplitDetailScreen.tsx`):
- Header with back button, split name, and DELETE button
- Stats bar showing exercise count and unique zone count
- FlatList displaying each exercise with name, zone badge, and sets x reps
- START WORKOUT button determines primary zone (most common in split)
- Generates session ID and navigates to WorkoutSession with splitId

**Navigation Types Update** (`src/navigation/types.ts`):
- Added optional `splitId` param to WorkoutSession
- Enables tracking split-sourced vs zone-sourced workouts

## Technical Decisions

| Decision | Rationale |
|----------|-----------|
| Primary zone = most common zone | Intuitive for multi-zone splits like "Push Day" |
| splitId optional param | Backward compatible with existing zone-based workflow |
| Delete but no edit in v1 | Simplifies scope - users can delete and recreate |
| Stats bar (exercises + zones) | Quick visual summary before starting |

## Data Flow

```
SplitDetailScreen
    |
    +-- useSplits().getSplitExercises(splitId)
    |       -> SplitExercise[] with exerciseId, exerciseName, primaryZone, defaultSets, defaultReps
    |
    +-- handleStartWorkout()
            1. Count zones -> { biceps: 2, shoulders: 3, abs: 1 }
            2. Sort by count -> primaryZone = "shoulders"
            3. navigation.navigate('WorkoutSession', {
                 sessionId, exercises: [...ids], zoneId: primaryZone, splitId
               })
```
