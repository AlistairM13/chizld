  <action>
Create useTempoVoice hook (src/hooks/useTempoVoice.ts):

Import Speech from expo-speech.

Interface TempoConfig:
- eccentric: number (seconds, default 3)
- pauseBottom: number (seconds, default 1)
- concentric: number (seconds, default 2)
- pauseTop: number (seconds, default 0)

Hook state:
- isActive: useRef<boolean>(false) to track if tempo cycle should continue
- isPlaying: useState<boolean> for UI indicator

Functions:
- speakAndWait(text: string): Promise<void> - Returns promise that resolves on Speech.speak onDone callback. Include onError to not block on failures.
- delay(ms: number): Promise<void> - setTimeout wrapper
- playTempoCycle(config: TempoConfig): async function that:
  1. Speaks "Eccentric"
  2. Counts down: 5, 4, 3, 2, 1 (or config.eccentric number)
  3. Speaks "Hold" if pauseBottom > 0, then delays
  4. Speaks "Concentric"
  5. Counts down: 3, 2, 1 (or config.concentric number)
  6. Speaks "Hold" if pauseTop > 0, then delays
  7. Loop while isActive.current === true
- startTempo(config: TempoConfig): Sets isActive.current = true, isPlaying = true, calls playTempoCycle
- stopTempo(): Sets isActive.current = false, calls Speech.stop(), sets isPlaying = false

Return { startTempo, stopTempo, isPlaying }

Key considerations from research:
- Always check isActive.current before each speak call to allow cancellation
- Use onDone callback, not polling isSpeakingAsync
- Call Speech.stop() in stopTempo to immediately halt any queued speech
  </action>

Hook compiles. startTempo speaks the sequence. stopTempo halts immediately.
  </verify>

  </done>
