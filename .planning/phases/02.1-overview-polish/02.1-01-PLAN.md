<task type="auto">
  <name>Task 2: Create PhotoSlot Component</name>
  <files>src/components/character/PhotoSlot.tsx</files>
  <action>
Create PhotoSlot component that displays an RPG equipment slot for zone progress photos.

Props interface:
```typescript
interface PhotoSlotProps {
  photoPath: string | null;  // URI to photo or null for placeholder
  isWarm: boolean;           // drives border style and colors
  size?: number;             // slot dimensions, default 50
  onPress?: () => void;      // for future tap handling
}
```

Default state (no photo):
- Square container: 50x50px (or size prop)
- Border radius: 2px (sharp corners)
- Centered "+" icon using JetBrains Mono Regular, 20px

Cold state (isWarm: false):
- Border: 1px dashed #555566 (colors.text.muted)
- "+" icon color: #555566
- No shadow/glow

Warm state (isWarm: true):
- Border: 1px solid #FF8C1A (colors.ember[500]) - NOT dashed
- "+" icon color: #FF8C1A
- Subtle ember glow: use shadowColor: colors.ember[500], shadowRadius: 6, shadowOpacity: 0.5, elevation: 4

Photo state (photoPath not null):
- Render Image with source={{ uri: photoPath }}
- resizeMode="cover"
- Same border style as warm/cold
- No "+" icon - image fills slot

Use Pressable for the container to support future onPress. Wrap with View for glow shadow (React Native shadow only works on View, not Pressable directly on Android).

Note on dashed border: React Native borderStyle: 'dashed' works on iOS but is inconsistent on Android. Use a workaround:
- For cold state, create a DashedBorder sub-component using multiple small View segments OR
- Accept solid border on Android as a fallback (visual polish, not blocking)
- Preferred: Use borderStyle: 'dashed' and document Android limitation
