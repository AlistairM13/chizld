New structure:
```typescript
interface ZoneCardProps {
  zone: ZoneWithIntensity;
  position: ZoneCardPosition;
  screenWidth: number;
  screenHeight: number;
}

export function ZoneCard({ zone, position, screenWidth, screenHeight }: ZoneCardProps) {
  // Calculate absolute position
  const cardX = position.x * screenWidth;
  const cardY = position.y * screenHeight;

  // Left-side zones: label left, slot right
  // Right-side zones: slot left, label right
  const isLeftSide = zone.side === 'left';

  return (
    <Pressable
      style={[
        styles.container,
        {
          left: cardX,
          top: cardY,
          flexDirection: isLeftSide ? 'row' : 'row-reverse',
        },
      ]}
      onPress={() => {
        // No-op for now - Phase 3 implements tap
      }}
    >
      <ZoneLabel
        zoneName={zone.name}
        level={zone.level}
        isWarm={zone.isWarm}
        side={zone.side}
      />
      <View style={styles.gap} />
      <PhotoSlot
        photoPath={null}  // No photos yet - future feature
        isWarm={zone.isWarm}
        size={50}
      />
    </Pressable>
  );
}
```

Styles:
```typescript
const styles = StyleSheet.create({
  container: {
    position: 'absolute',
    alignItems: 'center',
    // Transform to center on position point
    transform: [{ translateX: -65 }, { translateY: -25 }],
  },
  gap: {
    width: 8, // Space between label and slot
  },
});
```

Key points:
- No background, no border on container - transparent
- flexDirection: 'row' for left zones, 'row-reverse' for right zones
- ZoneLabel handles its own text alignment based on side prop
- PhotoSlot is always 50x50
- Total width ~130px (label ~72px + gap 8px + slot 50px)
- Container transform centers it on the position point

Part A: Update layout.ts

Add slotOffsetX to ZoneCardPosition interface to indicate where the PhotoSlot center is relative to the card position:

```typescript
export interface ZoneCardPosition {
  x: number;           // Card center X position (percentage of screen width)
  y: number;           // Card center Y position (percentage of screen height)
  anchorX: number;     // Anchor point on character body
  anchorY: number;     // Anchor point on character body
  slotOffsetX: number; // Offset from card center to PhotoSlot center (pixels)
}
```

For left-side zones (label left, slot right):
- slotOffsetX = +40 (slot is 40px to the right of card center)

For right-side zones (slot left, label right):
- slotOffsetX = -40 (slot is 40px to the left of card center)

Part B: Update ConnectingLines.tsx

Modify line path calculation to start from PhotoSlot center instead of card center:

```typescript
// Card center position
const cardX = positions.x * screenWidth;
const cardY = positions.y * screenHeight;

// Photo slot center (offset from card center)
const slotX = cardX + positions.slotOffsetX;
const slotY = cardY; // Same Y position

// Anchor position on body
const anchorX = positions.anchorX * screenWidth;
const anchorY = positions.anchorY * screenHeight;

// Create line path from PHOTO SLOT to anchor
const path = Skia.Path.Make();
path.moveTo(slotX, slotY);  // Start from slot, not card center
path.lineTo(anchorX, anchorY);
```

This makes the dashed lines visually connect the photo slot (the "equipment slot") to the corresponding body part.
