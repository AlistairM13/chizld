# Chizld Overview Screen — Polish + Card Redesign

I've attached a screenshot of the current overview screen in @progress-screenshots\post-phase2.png and also my original Figma design at @designs\overview.png. The current implementation merged the zone name and photo slot into one card, but they should be SEPARATE elements.

## Card Redesign — Most Important Change

The current card combines everything into one block. Instead, each zone should have TWO distinct elements side by side:

**Zone Label** (text block):
- Zone name: Barlow Condensed, 14px, uppercase
- Level: JetBrains Mono, 11px (e.g., "LV.5")
- No background, no border — just floating text
- Positioned to the LEFT of the photo slot (for left-side zones) or RIGHT of it (for right-side zones)

**Photo Slot** (square):
- Size: ~50×50px
- Default state (no photo): 1px dashed border in #555566, "+" icon centered in #555566
- When a photo exists: the image fills the slot, replacing the "+" entirely. The image should be cropped to fill (cover mode) with 2px border-radius
- The slot should look like an RPG equipment slot — an empty socket waiting to be filled

**Layout per zone:**
- Left-side zones (Traps, Biceps, Forearms, Tibialis): `[ZONE NAME / LV] [📷 slot]` — text left, slot right, with the dashed connecting line extending from the slot toward the character
- Right-side zones (Neck, Shoulders, Abs, Quads): `[📷 slot] [ZONE NAME / LV]` — slot left, text right, with the dashed line extending from the slot toward the character

This way, when a progress photo is taken after a workout and tagged to a zone, the small thumbnail appears in that zone's slot on the overview screen — like equipping an item in an RPG inventory. The zone name and level always remain visible beside it.

**Data flow for photos:**
- Each zone can have a `latest_photo_path` (nullable string)
- If `latest_photo_path` is set, render the image in the slot using `<Image source={{ uri: path }}` with `resizeMode="cover"`
- If null, show the "+" placeholder
- For now, all slots will show the "+" since no photos exist yet — but the component structure should support both states

## Polish Changes (Same as Before)

### 1. Zone Card Styling
- Remove any card background/container — the zone label text should float freely
- The photo slot is the only "card-like" element, and it should be minimal (dashed border only)
- Sharp corners on the slot: 2px border-radius max

### 2. Warm vs Cold Zone States
Show visual differentiation — hardcode TRAPS, BICEPS, SHOULDERS as "warm" for now:

**Warm (active) zones:**
- Zone name text: #FF8C1A (ember-orange)
- Level text: #FFB366 (light ember)
- Photo slot border: 1px solid #FF8C1A (not dashed, solid)
- Subtle ember glow on the slot: use elevation + ember shadow
- "+" icon in the slot: #FF8C1A

**Cold (untrained) zones:**
- Zone name text: #8888A0
- Level text: #555566
- Photo slot border: 1px dashed #555566
- No glow
- "+" icon: #555566

### 4. Tab Bar
- Background: #0A0A0F (matching main bg exactly)

- Active: #FF8C1A, Inactive: #555566

### 5. System Status Text
